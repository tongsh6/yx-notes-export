# 技能卡：PySide6 长任务交互体验 (Skill: PySide6 Long-Task UX)

## 适用场景
- 导出/同步/扫描等分钟级任务
- 用户反馈“卡住”“没反应”“进度不动”
- 需要在不破坏线程安全前提下提供可中断交互

## 输入
- 任务线程实现（`QThread`/worker）
- 网络或 I/O 调用链（重试、限流、超时）
- GUI 状态区域（状态栏、日志区、按钮）

## 标准做法
1. 定义活动信号：worker 在关键阶段发 `activity`（请求开始、等待重试、失败、收尾）
2. 主线程维护 watchdog：无活动超过阈值发提示（含最后状态）
3. 提供协作式操作：`停止`/`跳过当前`，并明确“等待当前请求返回后生效”
4. 禁止强杀线程：不使用 `terminate()`，避免状态损坏
5. 结构化日志落点：记录 `api_wait_retry`、`export.stall_hint`、`session_stop/skip`

## 验收标准
- 用户在任何时刻都能知道“程序在做什么”
- 长等待时可执行“停止或跳过当前”
- 任务结束后状态可收敛（按钮、文案、统计）
- 测试通过且无新增 LSP error

## 反模式
- 只更新进度条，不更新状态文案
- 仅在循环边界检查 abort，阻塞调用不可中断
- 线程强制终止
