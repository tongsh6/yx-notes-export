# Best Practice: GUI 长任务体验

## 原则
- 先可见，再可控：先解决“用户不知道系统在干什么”
- 状态优先于百分比：长任务场景里状态文案比进度数字更关键
- 协作取消优先：优先 stop/skip 标志位，不做线程强杀

## 实践
- 统一状态通道：网络层 -> worker -> UI
- 建立 stall 机制：阈值提醒 + 最后状态 + 操作入口
- 控件策略统一：中文按钮使用最小宽+弹性布局+统一高度
- 分层迭代：稳定性修复与视觉优化分批进行

## 禁止项
- 用 `terminate()` 处理“看起来卡住”
- 用固定宽度硬压全部按钮
- 只在日志输出错误，不在 UI 给可操作反馈
