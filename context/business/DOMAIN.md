# 业务域概览

## 目标
通过官方 API 连接印象笔记服务端，允许用户配置认证信息后，自动批量导出笔记为结构化 Markdown 文件。
导出目录严格按照印象笔记的原始层级（分组→笔记本→笔记）组织。

## 核心用户场景
1. **全量导出**：将账户下所有笔记一次性导出
2. **笔记本级导出**：指定一个或多个笔记本进行导出
3. **单笔记导出**：指定笔记 GUID 或标题导出
4. **增量导出**：只导出上次导出后新增/修改的笔记（CLI `--incremental` / GUI「仅增量」）

## 层级结构（印象笔记数据模型）
```
账户
└── 分组 (Stack)          # 笔记本的分组，可为空
    └── 笔记本 (Notebook) # 每条笔记归属一个笔记本
        └── 笔记 (Note)   # 正文 + 标签 + 元数据 + 附件
            └── 附件 (Resource)
```

## 输出目录结构
```
output/
└── {stack}/              # 无分组的笔记本直接在根目录
    └── {notebook}/
        ├── {note-title}.md
        └── assets/       # 该笔记本下所有笔记的附件
            └── {hash[:8]}-{filename}
```

## 笔记内容组成
- **标题**：作为 Markdown 一级标题 + 文件名
- **正文**：ENML（印象笔记 XML）→ Markdown 转换
- **标签**：输出到 YAML Front Matter
- **元数据**：创建时间、更新时间、来源 URL 等，输出到 Front Matter
- **附件**：图片内联嵌入，其他文件以链接引用

## 认证方式（关键决策点）

| 方式 | 用户体验 | 安全性 | 适用场景 |
|------|----------|--------|----------|
| Developer Token | 配置一次即可，Token 有效期约 1 年 | 中 | 个人工具，**优先实现** |
| 用户名 + 密码 | 最简单，无需额外操作 | 低（明文存储风险） | 按用户需求可扩展 |
| OAuth | 浏览器授权一次，Token 自动刷新 | 高 | 分发给他人的工具 |

> 注：evernote-backup 开源项目已验证印象笔记中国区支持用户名+密码登录。
> 初版优先实现 Developer Token，用户名+密码作为可选项。

## 关键元数据（需保留）
- 创建时间（created）
- 更新时间（updated）
- 来源 URL（source-url）
- 标签（tags）
- 笔记本归属（notebook name + stack）

## 文件命名规则
- 以笔记标题为文件名，替换非法字符（`/\:*?"<>|`）为 `-`
- 同名笔记追加序号（note-title-2.md）
- 附件使用 `{resource-hash[:8]}-{filename}` 命名，避免冲突
