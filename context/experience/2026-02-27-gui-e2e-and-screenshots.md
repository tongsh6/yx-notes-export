# GUI 端到端测试与截图规范

## 背景
需要可重复、可自动化的 GUI 端到端测试，同时避免测试截图散落到项目根目录。

## 现象
- 传统 GUI 验证只能手工执行，无法自动回归。
- 截图输出路径不统一，容易污染仓库。

## 结论
- 使用 `pytest-qt` 实现 GUI 端到端测试。
- 通过 Fake Worker 模拟连接与导出流程，保证离线可重复。
- 截图统一输出到 `tests/artifacts/screenshots/`，默认自动清理。
- 仅设置 `YX_SCREENSHOT=1` 时才产出截图，避免噪音。

## 影响
- GUI 测试可纳入 CI（离线模式）。
- 截图管理可控，不污染工作目录。
